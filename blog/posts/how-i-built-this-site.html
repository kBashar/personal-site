How I Built This Site

Feb 20, 2026
meta python web

A walkthrough of the tooling behind this portfolio: pure HTML/CSS, a 100-line Python build script, and GitHub Actions for deployment.

Every few years I rebuild my personal site from scratch. This time I wanted something I could fully understand, maintain without a framework, and deploy for free. Here's how it works.

Goals

  • No JavaScript framework. Vanilla HTML, CSS, and a sprinkling of JS for the nav toggle.
  • Markdown-based blog — I write posts in .md files, a build script converts them to HTML.
  • Deployable to GitHub Pages with a single git push.
  • Fast. No build step when loading the page, no runtime dependencies.

The Stack

Static HTML pages for everything that doesn't change frequently: home, about, portfolio, contact.

A Python build script (build.py) that reads Markdown files from posts/, injects them into an HTML template, and writes the result to blog/posts/. It also regenerates blog/index.html sorted by date.

GitHub Actions runs build.py on every push to main, then deploys the entire repo to GitHub Pages.

The Build Script

The script uses two libraries:

import yaml      # for YAML front matter
import markdown  # for converting .md bodies to HTML

Each post starts with YAML front matter:

---
title: "How I Built This Site"
date: "2026-02-20"
description: "..."
tags: ["meta", "python"]
---

The script splits on the --- delimiter, parses the YAML, converts the Markdown body, and fills an HTML template using simple {{placeholder}} substitution. No Jinja2, no template engine — just str.replace().

Total build time for a dozen posts: under a second.

CSS Architecture

All color tokens and spacing live in CSS custom properties at :root. A single data-theme="dark" attribute on <html> overrides the color tokens for dark mode. The toggle is about 10 lines of JS.

:root {
  --color-bg:     #F9F9F7;
  --color-text:   #111111;
  --color-accent: #0066CC;
}

[data-theme="dark"] {
  --color-bg:     #111111;
  --color-text:   #E0E0E0;
  --color-accent: #4D9FFF;
}

No framework, no preprocessor. Just custom properties and clamp() for fluid type.

Deployment

.github/workflows/deploy.yml does three things:

  1. Checks out the repo
  2. Installs deps and runs python build.py
  3. Deploys the directory as a GitHub Pages artifact

The generated HTML files are committed alongside source — so the site is always browsable directly from the repo, even if Actions is down.

What I'd Do Differently

If I were building this for a team or a client, I'd reach for something more established — Eleventy, Astro, or Hugo. For a personal site I own completely and touch infrequently, the simplicity of "it's just files" is worth the occasional manual work.

The total build tooling is about 150 lines of Python. I can read the whole thing in five minutes. That's the point.
